# Usuarios y roles V2.4

- La cuenta propietaria puede crear y editar usuarios aunque la lista de permisos aún no haya cargado.
- Roles predefinidos y permisos personalizados.
- Activar, desactivar, restablecer contraseña y eliminar acceso.
- Acciones maestras para deshabilitar todos y limpiar usuarios demostrativos antiguos.
- Mensajes visibles de éxito o error en cada operación.

Nota: eliminar el acceso borra el documento de `roy_users`. Para borrar también la cuenta de Firebase Authentication se debe hacer desde Authentication > Usuarios, porque el navegador no puede eliminar cuentas de otras personas de forma segura.
